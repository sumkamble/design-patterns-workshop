[[ch02-singleton]]
== Singleton Pattern

=== GoF Definition

Ensure a class only has one instance, and provide a global point of access to it.

=== UML Representation

[plantuml, singleton, png]
----
class Cache {
    {static} -Cache INSTANCE
    {static} Cache getInstance()
}
class Application
Cache *--> Cache
Application --> Cache
----

=== Singleton and reflection

(((Singleton, Reflection)))
Singleton objects can be instantiated using reflection and hence the objective can be overridden using reflection

=== Singleton and serialization

(((Singleton, Serialization)))
If the singleon class supports serialization then it can be serialized and deserialized to override the objective of maintaining one instance.
